HTML Semântico

Semântica

Na programação, a Semântica refere-se ao significado de um pedaço de código - por exemplo "que efeito tem a execução daquela linha de JavaScript?" ou "que propósito ou função esse elemento HTML tem" (em vez de "como é ?".) ​

Atualmente o papel da HTML não é apenas estruturar documentos para a web, mas também descrever o significado do conteúdo presente nesses documentos por meio de tags semânticas. O HTML semântico tem como objetivo descrever o significado do conteúdo presente em documentos HTML, tornando-o mais claro tanto para programadores quanto para browsers e outras engines que processam essa informação.

HTML (HyperText Markup Language) é um linguagem de marcação muito conhecida e utilizada na criação de páginas web. Ela renderiza informações nos navegadores de internet juntamente com o CSS, JavaScript e outras linguagens de desenvolvimento web, permitindo que nós tenhamos acesso a todo o conteúdo de um site.

O HTML é como o esqueleto que dará forma às páginas. É a estrutura mais básica. Ele setoriza as diversas partes da página de acordo com o seu conteúdo, ou seja, ajuda a endereçar cada um dos elementos que a constitui, como os textos, imagens, menu de navegação, links, dentre outros. Posteriormente ele irá auxiliar a aplicação de outras funcionalidades a um site.

Algumas tags HTML e seus Significados
  • <!DOCTYPE>: define o tipo de documento
  • <html>: define o início do documento HTML, todas as demais tags devem estar dentro desta tag
  • <head>: cabeçalho onde ficam os metadados do documento, como meta tags, CSS e JavaScript
  • <title>: define o título do documento, mostrado na barra de títulos do navegador e como link nos mecanismos de busca
  • <base>: define a URL base do documento
  • <link>: utilizado para linkar um arquivo de imagem ou CSS à página HTML
  • <script>: utilizado para escrever ou anexar um arquivo normalmente JavaScript à página
  • <meta>: meta tags, como charset, description, viewport ou keywords
  • <style>: utilizada para adicionar um código CSS à página
  • <body>: corpo ou área principal do documento, dentro deste deve ficar o conteúdo da sua página
  • <header>: define um cabeçalho da página ou de uma seção, normalmente pode conter um menu (nav), um logo ou link
  • <nav>: define uma área de navegação contendo links, para a formação de menus com hiperlinks
  • <footer>: define o rodapé de uma página ou seção
  • <main>: define o conteúdo principal da página, deve ser utilizado apenas uma única vez em cada página
  • <section>: define uma seção do documento
  • <article>: pode ser um post ou notícia de um fórum ou blog, define um conteúdo independente
  • <h1> a <h6>: representam 6 níveis de títulos, sendo o <h1> de maior importância
  • <p>: define um parágrafo
  • <br>: define uma quebra de linha
  • <div>: define uma camada sem formatação, genérico
  • <ul>: define uma lista não ordenada
  • <li>: define um elemento ou item de uma <ul> ou <ol>
  • <a>: define um link que deve ser apontado com o atributo href
  • <img>: define a utilização de uma imagem (JPG, GIF, PNG e outras). A imagem deve ser apontada com o uso do atributo src
  • <strong>: apesar de deixar em negrito, representa a importância deste texto no meio da frase
  • <span>: utilizado para definir pequenos trechos de textos
  • <pre>:exibe o texto exatamente da forma que foi digitado, incluindo tabulações, espaços e enters
Conhecendo os elementos semânticos do HTML5

<meta>

A <meta> ajuda o seu site a descrever seu conteúdo aos buscadores, como o Google. Aqui você poderá dizer qual a codificação (charset), descrição e até palavras-chaves referentes a sua página. Essas são importantes justamente por identificar o site aos buscadores, ajudando no SEO (Search Engine Optimization). O Google não utiliza somente as metas para realizar suas buscas, mas elas são uma parte importante.

  • description: guarda a descrição do site, utilizado nos motores de busca.
  • keywords: palavras chaves relacionadas ao site, utilizada nos primórdios pelos motores de busca.
  • author: identifica o autor do site
  • robots: ajuda os robôs dos motores de busca a indexar ou não um site. Com esta opção podemos dizer se uma determinada página do site não deve ser mostrada nos resultados de uma busca.
Exemplo de código: 
    
      <head>
      <title>Página do Burnes - Home </title>
      <meta charset="utf-8">
      <meta name="description" 
      content="Página do Burnes sobre HTML e CSS">
      <meta name="keywords" 
      content="html, css, js, site">
      <meta name="robots" content="index">
      </head>

<nav>

O elemento <nav> representa um menu de navegação na sua página. Normalmente é o primeiro elemento da nossa página, mas independentemente da posição que você escolher para a sua navbar é boa prática envolver o agrupamento de links com esta tag.

Exemplo de código: 
         
          <nav>
           <ul>
           <li> <a> href="”#”">pagina 1</a> </ul>
           <li> <a> href="”#”">pagina 2</a> </ul>
           <li> <a> href="”#”">pagina 3</a> </ul>
           <li> <a>href="”#”">pagina 4 </a> </ul>
           </ul>
           </nav>

<section>

O elemento <section> representa uma seção dentro de um documento e geralmente contém um título, o qual é definido por meio de um dos elementos entre <h1> e <h6>. Podemos utilizar o <section> , por exemplo, para descrever as seções/tópicos de um documento.

Exemplo de código: 
      
        <section>
        <h3> Seção 1 </h3>
        <p>Lorem ipsum dolor sit amet,
       consectetur adipisicing elit,
       sed do eiusmod ...
        </p>
        </section>

<iframe>

O elemento HTML <iframe> representa um contexto de navegação aninhado, incorporando outra página HTML na atual. iFrame é a abreviação de Inline Frame. É um elemento poderoso quando se trata de web design. Você provavelmente já viu diversos vídeos do YouTube que estavam incorporados em outros sites. Para além disso, um iFrame pode inserir todo tipo de mídia.

Atributos:
  • O iFrame source (src) é a origem do conteúdo, seja do servidor interno o externo. Não se esqueça de incorporar o código na url.
  • Width (largura) e height (altura) são os aspectos visuais. Você pode inserir uma medida fixa como 680×480 pixels (px) no exemplo. Ou, você pode usar o método de porcentagem (10%-100%) na qual o conteúdo será ajustado automaticamente.
Exemplo de código: 
    
      <iframe>
       id="inlineFrameExample"
       title="Inline Frame Example"
       width="300"
       height="200"
       src="url.link">
      </iframe>

<a>

O elemento HTML <a> (ou elemento âncora), com seu atributo href, cria um hiperlink para páginas da web, arquivos, endereços de e-mail, locais na mesma página ou qualquer outra coisa que uma URL possa endereçar.

Exemplo de código: 
    
      <p>You can reach Michael at:</p>
      <ul>
      <li>
       <a 
       href="https://example.com">
       Website
       </a>
      </li>
      <li>
       <a href="tel:+123456789" >Phone</a>
      </li>
      </ul>

CSS

O que é CSS?

CSS (Cascading Style Sheets ou Folhas de Estilo em Cascata) é uma linguagem de estilo (en-US) usada para descrever a apresentação de um documento escrito em HTML ou em XML (incluindo várias linguagens em XML como SVG, MathML ou XHTML). O CSS descreve como elementos são mostrados na tela, no papel, na fala ou em outras mídias.

CSS (Cascading Style Sheets) é o código que estiliza o conteúdo da web. Noções básicas de CSS explicam o que você precisa para começar. Responderemos perguntas como: Como faço para deixar o texto vermelho? Como faço para exibir o conteúdo em um determinado local no layout (página da web)? Como posso decorar minha página da web com imagens e cores de fundo? ​

Assim como o HTML, o CSS não é uma linguagem de programação. Também não é uma linguagem de marcação. CSS é uma linguagem de folha de estilos. CSS é o que você usa para estilizar elementos HTML seletivamente. Por exemplo, este CSS seleciona o texto do parágrafo, definindo a cor como vermelho:

  • A propriedade CSS color define a cor do texto a ser usada.
  • A propriedade CSS font-family define a fonte a ser usada.
  • A propriedade CSS font-size define o tamanho do texto a ser usado.
Exemplo: 
    
    p {
       color: blue;
       font-family: verdana;
       font-size: 300%;
      }
Seletores CSS

Os seletores CSS são usados para definir um padrão dos elementos que você deseja selecionar para aplicar um conjunto de regras CSS nos elementos selecionados. Os combinadores definem o relacionamento entre os seletores. Usando vários seletores e combinadores, você pode selecionar e estilizar com precisão os elementos desejados com base em seu tipo, atributos, estado ou relacionamento com outros elementos.

Existem mais de 80 seletores e combinadores. Os seletores CSS podem ser agrupados nas seguintes categorias com base no tipo de elementos que podem selecionar.

O seletor de tipo seleciona todos os elementos que possuem o nome de nó fornecido. Por exemplo, div selecionará todos os elementos <div> e input corresponderá a qualquer elemento <input> . O seletor universal, indicado com um asterisco (*), é um seletor de tipo especial que seleciona todos os elementos.

O seletor de classe seleciona todos os elementos que possuem o atributo de classe determinado indicado pelo nome da classe prefixado com um ponto final (.). Por exemplo, .index corresponderá a qualquer elemento que possua class="index". O seletor de ID seleciona um elemento com base no valor de seu atributo id. O seletor é o id prefixado com um "sinal numérico" (U+0023, #). Por exemplo, #toc corresponderá ao elemento que possui id="toc". Classe e id são atributos globais. Deve haver apenas um elemento com um determinado id em um documento; mas se houver mais de um, o seletor de ID corresponderá a todos os elementos com esse ID.

Exemplo: 
    
* {
  font-style: italic;
}
p {
  color: red;
}
.myClass {
  text-decoration: underline;
}
#myId {
  font-family: monospace;
}
p.myClass#myId {
  font-size: 1.5rem;
}
Combinadores e separadores

  • > (Combinador filho)
  • || (Combinador de colunas) Experimental
  • ~ (combinador de irmãos subsequente)
  • " " (Combinador descendente)
  • | (Separador de namespace)
Seletores de atributos

O seletor de atributos CSS combina elementos com base no elemento que possui um determinado atributo definido explicitamente, com opções para definir um valor de atributo ou correspondência de valor de substring.

Os seletores de atributos selecionam todos os elementos que, dependendo de como o seletor é escrito, possuem o atributo fornecido ou possuem o atributo fornecido com uma correspondência de valor de substring. Por exemplo, [type] corresponderá a todos os elementos que possuem o atributo type definido (para qualquer valor) e [type="submit"] corresponderá a <input type="submit"> e <button type="submit"> , ou qualquer elemento com type="submit" definido, mesmo que este par atributo-valor seja suportado apenas em elementos<input> e <button> . A correspondência não diferencia maiúsculas de minúsculas.

[attr]
Representa elementos com um nome de atributo attr.
[attr=valor]
Representa elementos com um nome de atributo attr cujo valor é exatamente value.
[attr~=valor]
Representa elementos com um nome de atributo attr cujo valor é uma lista de palavras separadas por espaços em branco, uma das quais é exatamente value.
[attr|=valor]
Representa elementos com um nome de atributo attr cujo valor pode ser exatamente valor ou pode começar com valor imediatamente seguido por um hífen, - (U+002D). É frequentemente usado para correspondências de subcódigos de idioma.
[attr^=valor]
Representa elementos com um nome de atributo attr cujo valor é prefixado (precedido) por valor.
[attr$=valor]
Representa elementos com um nome de atributo attr cujo valor é sufixado (seguido) por valor.
[attr*=valor]
Representa elementos com um nome de atributo attr cujo valor contém pelo menos uma ocorrência de valor na string.
[attr operador e valor i]
Adicionar um i (ou I) antes do colchete de fechamento faz com que o valor seja comparado sem distinção entre maiúsculas e minúsculas (para caracteres dentro do intervalo ASCII).
[attr operador e valor s] Experimental
Adicionar um s (ou S) antes do colchete de fechamento faz com que o valor seja comparado com distinção entre maiúsculas e minúsculas (para caracteres dentro do intervalo ASCII).
Exemplo: 
    
a[title] {
  color: purple;
}

a[href="https://example.org"]
{
  color: green;
}
a[href*="example"] {
  font-size: 2em;
}
a[href$=".org" i] {
  font-style: italic;
}
a[class~="logo"] {
  padding: 2px;
}
Pseudo-classes e Pseudo-elementos

Pseudo-classes

Uma pseudo-classe CSS é uma palavra-chave adicionada a um seletor que especifica um estado especial do(s) elemento(s) selecionado(s). Por exemplo, a pseudo-classe :hover pode ser usada para selecionar um botão quando o ponteiro do usuário passa sobre o botão e esse botão selecionado pode então ser estilizado.

Uma pseudo-classe consiste em dois pontos (:) seguidos pelo nome da pseudo-classe (por exemplo, :hover). Uma pseudo-classe funcional também contém um par de parênteses para definir os argumentos (por exemplo, :dir()). O elemento ao qual uma pseudo-classe está anexada é definido como um elemento âncora (por exemplo, botão no caso de botão: passar o mouse).

As pseudo-classes permitem aplicar um estilo a um elemento não apenas em relação ao conteúdo da árvore do documento, mas também em relação a fatores externos como o histórico do navegador (:visited, por exemplo), o status do seu conteúdo ( como :checked em certos elementos do formulário) ou a posição do mouse (como :hover, que permite saber se o mouse está sobre um elemento ou não).

Pseudo-elementos

Um pseudo-elemento CSS é uma palavra-chave adicionada a um seletor que permite estilizar uma parte específica do(s) elemento(s) selecionado(s).

Dois pontos duplos (::) são usados para pseudo-elementos. Isso distingue pseudo-elementos de pseudo-classes que usam dois pontos (:) em sua notação.

Você pode usar apenas um pseudo-elemento em um seletor. O pseudo-elemento deve aparecer depois de todos os outros componentes do seletor complexo ou composto em que aparece. Por exemplo, você pode selecionar a primeira linha de um parágrafo usando p::first-line, mas não os filhos da primeira linha ou uma primeira linha suspensa. Portanto, ambos p::first-line > * e p::first-line:hover são inválidos.
